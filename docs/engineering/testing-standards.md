# Testing Standards

## Princípios
- Prefira testes próximos ao comportamento de negócio.
- Cubra caso principal, erro e edge cases relevantes.
- Todo bug corrigido deve ganhar teste de regressão quando viável.
- Refatoração relevante pede segurança por testes.

## Mínimo esperado
- Feature nova: testes ou critérios verificáveis claros.
- Bugfix: reprodução + teste de regressão.
- Refactor: cobertura suficiente para garantir ausência de mudança comportamental.
