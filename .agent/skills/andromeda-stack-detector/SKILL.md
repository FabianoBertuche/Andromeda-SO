# Objetivo
Antes de alterar o projeto, identificar a stack e os componentes impactados.

# Instruções
1. Detecte linguagem principal, framework, gerenciador de pacotes e estrutura do projeto.
2. Identifique padrões arquiteturais já existentes.
3. Liste arquivos e módulos prováveis afetados.
4. Não proponha solução antes de entender o estilo atual do repositório.

# Saída esperada
- stack detectada
- área impactada
- riscos de compatibilidade
- workflow recomendado
