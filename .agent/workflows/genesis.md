# Genesis

## Use quando
- nova feature importante
- novo módulo
- mudança de contrato
- alteração estrutural

## Passos
1. Entender objetivo de negócio.
2. Fazer análise de contexto atual.
3. Passar por `andromeda-architecture-guard`.
4. Escrever plano com `writing-plans`.
5. Definir testes esperados.
6. Implementar incrementalmente.
7. Rodar `andromeda-rule-check`.
8. Finalizar com `andromeda-definition-of-done`.
